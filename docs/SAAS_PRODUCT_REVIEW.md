# SaaS 상용화 검토

작성일: 2026-05-22 KST

## 결론

현재 앱은 MVP의 방향은 좋지만, 상용 SaaS로 보이려면 단순히 기능을 더 붙이는 것보다 `완성되는 예배 PPT`까지 가는 주 흐름을 더 단단하게 만들어야 한다.

가장 중요한 차별점은 일반 발표 도구가 아니라 `예배 순서와 슬라이드 제작이 한 화면에서 연결되는 도구`라는 점이다. 이 방향은 유지한다.

## 벤치마킹 요약

### Planning Center Services

- 예배 순서, 봉사자, 파일, 리허설 자료를 한 플랜 안에 묶는다.
- 순서 항목을 줄 단위로 추가하고 드래그로 재정렬한다.
- 파일 첨부, 색상 구분, 공유가 핵심이다.
- 우리 앱에 적용할 점: 예배 순서 항목과 슬라이드가 1:1로 이어지는 구조, 항목 유형(찬양/말씀/광고/기도/헌금) 제공.

### Beautiful.ai

- Smart Slides, 브랜드 테마, 템플릿, 협업, 권한, 분석, PPT/Google Slides 내보내기가 핵심이다.
- 사용자가 디자인을 망치지 않도록 레이아웃 규칙이 자동으로 잡힌다.
- 우리 앱에 적용할 점: 예배용 Smart Layout, 기본 템플릿, 폰트/여백/배경 프리셋, PPTX export 품질.

### Canva / Pitch / Prezi 계열

- 템플릿 갤러리, 브랜드 키트, 에셋 라이브러리, 협업, 공유 링크가 상용 제품 느낌을 만든다.
- 고급 편집보다 시작 템플릿과 결과물 신뢰도가 중요하다.
- 우리 앱에 적용할 점: 교회별 브랜드 키트, 최근 프로젝트, 빠른 복제, 공유/다운로드 플로우.

## 현재 앱의 강점

- 예배 순서와 PPT 편집을 한 화면에 묶은 방향이 좋다.
- 한국어 예배 준비 흐름에 맞춘 가사/말씀 추가 기능이 이미 있다.
- `Agents.md` 기준으로 기능 경계가 분명해서 계속 확장하기 쉽다.
- 로컬 상태 저장, undo, 드래그 순서 변경, 텍스트 분할 등 MVP 핵심이 있다.

## 상용화 전 필수 보강

### 1. 핵심 결과물

- PPTX 내보내기
- PDF 내보내기
- 출력 전 전체 미리보기
- 폰트 깨짐/비율 깨짐 방지
- 다운로드 실패/완료 상태 표시

### 2. 예배 특화 워크플로우

- 예배 순서 항목 유형: 찬양, 말씀, 설교, 기도, 광고, 헌금, 축도
- 찬양 가사 붙여넣기 후 자동 슬라이드 생성
- 말씀 구절 전체 DB 연결
- 빈 슬라이드, 너무 긴 가사, 누락된 제목 자동 점검
- 지난 예배 복제해서 이번 주 예배 만들기

### 3. 편집기 품질

- 요소 리사이즈 핸들
- 도형/텍스트/이미지 복사, 삭제, 정렬
- 이미지 파일 업로드
- 레이어 순서 조정
- 키보드 단축키: delete, ctrl/cmd+c, ctrl/cmd+v, 방향키 이동
- 실제 PPT와 가까운 썸네일 렌더링

### 4. SaaS 기본 구조

- 로그인
- 프로젝트 목록
- 교회/팀 워크스페이스
- 자동 저장 서버 동기화
- 프로젝트 공유 링크
- 권한: owner, editor, viewer
- 결제: 무료/프로/교회팀 플랜

### 5. 신뢰와 운영

- 에러 경계 UI
- 로딩/저장/실패 상태
- 데이터 백업
- 개인정보 처리방침/이용약관
- 저작권 안내: 가사/악보/성경 역본
- 사용 로그와 오류 로그

## 추천 개발 순서

### Phase 1: MVP를 실제 사용 가능하게

1. PPTX export
2. 실제 파일 업로드 이미지
3. 요소 리사이즈/삭제/복사
4. 전체 미리보기와 품질 점검
5. 예배 순서 항목 유형

### Phase 2: SaaS 제품처럼 보이게

1. 프로젝트 대시보드
2. 새 예배 만들기/지난 예배 복제
3. 템플릿 갤러리
4. 브랜드 키트
5. 온보딩 체크리스트

### Phase 3: 서버형 SaaS

1. 인증
2. DB 저장
3. 팀 워크스페이스
4. 공유 링크
5. 결제/구독

## 다음 구현 제안

바로 다음 작업은 `PPTX export + 전체 미리보기 + 품질 점검`이 가장 좋다. 사용자가 앱을 써보고 "진짜 파일이 나온다"를 경험해야 제품 가치가 생긴다.

그 다음은 `프로젝트 대시보드`다. SaaS 첫 화면은 편집기가 아니라 최근 예배/새 예배/템플릿을 보여주는 작업 허브가 되어야 한다.

## 외부 LLM 사용 판단

지금은 다른 LLM에게 넘기기보다 이 저장소 안에서 제품 방향과 구현 로그를 계속 쌓는 편이 낫다.

외부 LLM이 도움이 되는 경우:

- 교회 사역자 인터뷰 질문지 만들기
- 랜딩페이지 카피 후보 뽑기
- 가격 정책 후보 비교
- 성경/찬양 저작권 리서치 초안 만들기

외부 LLM에게 맡기지 않는 게 좋은 경우:

- 코드 구조 결정
- 편집기 UX 구현
- 상태 관리 설계
- PPTX export 품질

이 부분은 현재 코드 흐름을 아는 쪽에서 계속 이어가는 게 안전하다.
